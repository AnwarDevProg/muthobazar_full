Final target architecture
1. Left drawer should become data-driven

Instead of hardcoded drawer items, we create a catalog like:

MBAdvancedElementCatalog

Each element item should define:

class MBAdvancedElementItem {
  final String groupId;        // title, subtitle, media, price, promo...
  final String elementType;    // title, media, price, cta...
  final String variantId;      // title_text_bold, media_circle_ring...
  final String binding;        // product.titleEn, product.thumbnailUrl...
  final String label;          // Product title EN
  final Map<String, dynamic> defaultStyle;
  final MBInspectorControlSet inspectorControls;
}

Then the drawer can render every group automatically.

4. Product dialog preview data

The Studio should receive a preview context, not just a raw product.

Target:

class MBAdvancedCardPreviewContext {
  final MBProduct product;
  final MBBrand? brand;
  final MBCategory? category;
  final MBProductVariation? selectedVariation;
  final MBProductPurchaseOption? selectedPurchaseOption;
}

Then both drawer preview and canvas preview can resolve real values from the current form state.

This means while editing a product, the left drawer item preview should show:

Actual product title
Actual product price
Actual thumbnail
Actual brand/category
Actual sale/discount

not generic demo text.

7. Clean V3 JSON save rule

Saved JSON should contain design instructions only.

Good JSON:

{
  "version": 20,
  "type": "muthobazar_card_design_advanced_v2",
  "templateId": "hero_poster_circle_diagonal_v1",
  "designFamilyId": "hero_poster_circle",
  "layout": {
    "cardWidth": 185,
    "cardHeight": 255,
    "borderRadius": 0,
    "resizeMode": "responsive",
    "lockElementsToCard": true,
    "cardLayoutType": "hero_poster_circle_diagonal_v1"
  },
  "palette": {},
  "nodes": [
    {
      "id": "title_01",
      "elementType": "title",
      "variantId": "title_text_bold",
      "binding": "product.titleEn",
      "visible": true,
      "locked": false,
      "position": {},
      "size": {},
      "style": {}
    }
  ],
  "metadata": {
    "cardLayoutType": "hero_poster_circle_diagonal_v1"
  }
}

Avoid saving runtime-only values like:

resolved product title
resolved price text
temporary preview state
selected product object
brand/category full object
debug fields

Only save label when the user intentionally overrides text.
